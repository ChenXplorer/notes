## 关于 · About

这里收录的是**长篇深度文章**——单篇通常在一万字到十万字之间，聚焦一个技术话题，做尽量完整的拆解。不发推特式短文、不发日常笔记、不发没想清楚的草稿。

This repo hosts **long-form deep-dives** — typically 10k to 100k characters per piece, each fully dedicated to a single technical topic. No micro-posts, no daily notes, no half-baked drafts.

### 写作风格 · Writing style

- **以事件为锚点**：每个技术论点都绑定一个真实的案例、数字、源码行号或公开事件 · Every claim is anchored to a concrete case, number, source line, or public incident
- **优先讲失败**：行业里大家都爱讲成功，但失败教的更多 · Failures teach more than successes
- **拒绝营销腔**：不写「下一代」「革命性」「颠覆」这种词 · No hype, no buzzwords
- **承认不确定**：能查到的给出处，查不到的明说是推断 · Cite when possible; mark speculation as speculation

### 更新频率 · Update cadence

不定期。一般一到两个月一篇。**质量优先于频率。**
Irregular. Roughly one article every 1–2 months. **Quality over velocity.**

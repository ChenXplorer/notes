# 开源项目架构设计文档索引

> [!abstract] 用途
> 这页用于把同目录下的开源项目架构文档串成一组可导航的 Obsidian 知识地图，便于按阅读路径、设计取向和执行边界横向比较。

## 推荐阅读路径

1. [[codex-learning-technical-guide-v2|Codex 运行机制学习手册]]：先建立 agent loop、tool call、sandbox、approval、Local / Remote / Cloud 的基础心智模型。
2. [[openclaw-architecture-learning-guide|OpenClaw 架构设计学习手册]]：理解 Gateway-first 的长驻个人助手控制面，以及消息入口、工具策略、远程节点和 sandbox 的关系。
3. [[hermes-architecture-learning-guide|Hermes Agent 架构设计学习手册]]：对照 OpenClaw，理解 AIAgent-first 的自改进 runtime、记忆、技能和 terminal backend。
4. [[vibe-trading-technical-architecture-guide|Vibe-Trading 技术实现与架构设计学习指南]]：看一个垂直领域 AI 系统如何把 agent、金融数据、回测、MCP 和 Web/API 组合起来。

## 横向对比

| 维度 | Codex | OpenClaw | Hermes Agent | Vibe-Trading |
|---|---|---|---|---|
| 系统定位 | Coding agent runtime | 个人 AI 助手控制面 | 自改进 agent runtime | 金融研究与回测工作空间 |
| 架构中心 | runtime / agent loop | 长驻 Gateway | `AIAgent` | session + agent + 金融领域工具 |
| 典型入口 | CLI、IDE / Desktop、Remote、Cloud | 消息平台、CLI、Web、Cron、Node | CLI、Gateway、Cron、ACP、API、Python library | CLI、Web/API、MCP、Docker |
| 工具执行边界 | sandbox mode + approval policy | tool policy + sandbox backend + node capability | `terminal.backend` 决定 local / Docker / SSH 等 | Python 工具、回测 runner、MCP、Docker 和路径配置 |
| 主要状态 | session、workspace、Cloud task | sessions、channels、auth、workspace、agent state | memory、skills、SQLite sessions、prompt cache | runs、sessions、JSON/JSONL、SQLite FTS、memory、`.env` |
| 主要风险 | 文件/命令越界、误删、过宽权限 | 消息入口授权、工具策略、sandbox bind mount | 默认 local 后端无隔离、长期记忆污染、插件/MCP 信任 | 生成代码执行、数据源错误、密钥泄漏、金融结论误用 |

## 复盘问题

- 模型提出 tool call 后，哪个运行时组件真正执行工具？
- 这套系统的状态保存在哪里，哪些状态会跨会话复用？
- sandbox 或执行后端保护的到底是什么，不保护什么？
- 技能、插件、MCP、内置工具分别处在哪个扩展层？
- 如果要从个人使用扩展到多人或生产环境，第一批风险边界在哪里？
